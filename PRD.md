# 啡点集 — 产品需求文档（PRD）

> 微信小程序 · 面向咖啡爱好者的个人打卡、收藏与分享工具

---

## 1. 产品概述

### 1.1 产品定位

「啡点集」是一款面向咖啡爱好者的微信小程序，帮助用户记录每日咖啡消费、可视化消费足迹、收藏咖啡美照，并生成精美分享卡片。

### 1.2 目标用户

- 日常饮用咖啡的年轻白领
- 喜欢探店和记录生活的咖啡爱好者
- 希望用数据和图片沉淀咖啡体验的用户

### 1.3 核心价值

| 价值维度 | 描述 |
|----------|------|
| 记录 | 一杯一打卡，完整记录咖啡日常 |
| 发现 | 地图足迹可视化，回顾去过的每家咖啡店 |
| 收藏 | AI 抠图咖啡架，打造个人咖啡美学陈列 |
| 分享 | 自动生成精美卡片，一键分享咖啡故事 |

| F1.4 | 月度统计 | 展示当月总杯数和去过的店铺数 |

### 3.2 咖啡打卡

**入口**：从打卡日历页点击「今日打卡」或「编辑」

#### 功能列表

| 编号 | 功能 | 描述 |
|------|------|------|
| F2.1 | 来源选择 | 二选一：「自制」或「消费」 |
| F2.2 | 店铺填写 | 消费模式下必填，支持手动输入（≤40 字） |
| F2.3 | 地图选点 | 调用微信选点能力，自动回填店铺名和经纬度 |
| F2.4 | 心情选择 | 5 个 emoji 候选（😀 😌 ☕️ 😴 😫） |
| F2.5 | 星级评分 | 1-5 星点选 |
| F2.6 | 心情记录 | 多行文本输入（≤300 字），支持换行 |
| F2.7 | 照片上传 | 从相册选择（≤6 张）或拍照（≤3 张），总上限 9 张 |
| F2.8 | 照片备注 | 每张照片可添加一句备注（≤30 字） |
| F2.9 | 删除照片 | 点击 × 号移除已选照片 |
| F2.10 | 提交打卡 | 保存到本地 + 异步同步到云端，防重复提交 |
| F2.11 | 编辑模式 | 加载已有数据进行修改，保留原始 ID 和创建时间 |

#### 数据模型

```
CoffeeLog {
  id: number              // 唯一标识
  date: string            // YYYY-MM-DD
  source: '自制' | '消费'
  shop: string            // 店铺名
  location: {             // 可选，消费模式下地图选点
    latitude: number
    longitude: number
    address: string
  }
  mood: string            // emoji
  rating: number          // 1-5
  note: string            // 心情记录
  photos: [{
    url: string           // cloud:// fileID
    cutoutUrl: string     // 抠图 fileID（可选）
    remark: string        // 照片备注
  }]
  createdAt: number       // 创建时间戳
}
```

#### AI 抠图流程

```
用户照片 (cloud://)
  → 云函数 segmentImage 下载原图
  → 上传到腾讯 COS + 触发 AIPicMatting
  → 获取去背景 PNG
  → 回传到微信云存储 (cutout/{openid}/)
  → 更新本地和云端数据
```

## 4. 技术架构

### 4.1 整体架构

```
┌─────────────────────────────────────────────┐
│                  前端（小程序）                │
│  ┌──────────┐ ┌──────────┐ ┌──────────────┐ │
│  │ 5 个页面  │ │ 自定义组件 │ │  工具模块     │ │
│  │ + Tab Bar │ │cached-img│ │ cloudStore   │ │
│  │          │ │          │ │ imageCache   │ │
│  │          │ │          │ │ cloudStorage │ │
│  │          │ │          │ │ date         │ │
│  └──────────┘ └──────────┘ └──────────────┘ │
└─────────────────┬───────────────────────────┘
                  │
    ┌─────────────┼─────────────┐
    ▼             ▼             ▼
┌────────┐  ┌─────────┐  ┌──────────┐
│ 云数据库 │  │ 云存储   │  │ 云函数    │
│user_data│  │ coffee/ │  │getOpenId │
│(JSON)   │  │ cutout/ │  │segment   │
│         │  │         │  │Image     │
└────────┘  └─────────┘  └──────────┘
                              │
                              ▼
                     ┌────────────────┐
                     │ 腾讯数据万象 CI  │
                     │ AIPicMatting   │
                     └────────────────┘
```

### 4.2 数据同步策略

```
本地优先（Local-First）架构：

1. 读取：先同步读 wx.getStorageSync → 立即渲染首屏
2. 拉取：异步调用 getJsonCached（15s TTL）获取云端最新数据
3. 合并：mergeCoffeeLogs 保留本地 cutoutUrl 和本地独有条目
4. 写入：wx.setStorageSync + 异步 setJson 双写
5. 冲突：以云端为主，但保留本地 cutoutUrl 等增量字段
```

### 4.3 图片缓存策略

```
cloud:// URL → imageCache 流程：

1. getSync：同步查内存 map → 命中直接返回本地路径（首帧无闪）
2. get：异步下载 → copyFile 到 USER_DATA_PATH/img/{hash}
3. LRU：最多 80 条，超限删最久未用；持久化到 Storage
4. prefetch：并发 4 路预下载首屏前 24 张
5. inflight：同一 fileID 去重，防并发下载
```

### 4.4 安全与稳定性

| 措施 | 说明 |
|------|------|
| 输入净化 | normalizeText 去除控制字符、截断超长、压缩空白 |
| Key 校验 | 云数据库 key 正则校验 + 原型链注入防护 |
| 请求去重 | 各页面 _loadReqId / _shelfReqId 防并发覆盖 |
| 防抖刷新 | 后台抠图完成后 500ms 防抖合并多次 loadShelf |
| 云端回退 | setJson 失败 → 读取合并后 set → 区分"文档不存在"和"网络错误" |
| 存储容量 | imageCache 持久化失败时自动淘汰半数缓存 |
| 匿名上传 | openid 获取失败时不阻塞上传，使用 anonymous 前缀 |
| 时区安全 | parseLocalDate 安全解析日期字符串，避免 UTC/本地时区偏移 |

## 5. 数据存储

### 5.1 云数据库（`user_data` 集合）

以用户 openid 为文档 ID，包含以下字段：

| Key | 类型 | 描述 |
|-----|------|------|
| `coffeeLogs` | Object | 以日期 `YYYY-MM-DD` 为 key 的打卡记录 map |
| `shelfItems` | Array | 用户手动添加到咖啡架的照片列表 |
| `shelfHiddenLogIds` | Array | 在咖啡架上隐藏的打卡照片 ID 列表 |

### 5.2 云存储

| 路径 | 内容 |
|------|------|
| `coffee/{openid}/` | 用户上传的原始照片 |
| `cutout/{openid}/` | AI 抠图后的 PNG 结果 |

### 5.3 本地存储

| Key | 内容 |
|-----|------|
| `coffeeLogs` | 打卡数据本地镜像 |
| `shelfItems` | 咖啡架数据本地镜像 |
| `shelfHiddenLogIds` | 隐藏 ID 本地镜像 |
| `imageCacheMap` | 图片缓存映射 `{ map, list }` |
