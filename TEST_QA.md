# 咖啡打卡小程序 - QA 自测清单

## 一、本次自测发现并已修复的问题

| # | 问题 | 处理 |
|---|------|------|
| 1 | **咖啡架一次添加多张图时 id 可能重复**：`addFromAlbum` 里用 `Date.now()` 生成 id，同一批多张会得到相同时间戳，id 可能重复。 | 改为 `shelf_${base}_${i}_${random}`，保证同批内唯一。 |
| 2 | **打卡页重复点击「今日打卡」可能产生两条记录**：无防重复提交，快速双击会触发两次 onPunch。 | 增加 `punching` 状态：onPunch 开头检查并置 true，保存失败时重置，成功清空表单时重置；按钮 `disabled="{{punching}}" loading="{{punching}}"`。 |

## 二、核心功能自测用例（建议真机/开发者工具执行）

### 1. 照片与云存储

| 用例 | 操作 | 预期 |
|------|------|------|
| 1.1 打卡页添加照片 | 打卡页 → 添加照片 → 选 1～多张 | 出现「上传中…」，上传成功后照片出现在列表，每张底部有「添加备注」输入框。 |
| 1.2 备注输入 | 在任一张照片底部输入备注 | 输入实时生效，保存打卡后该条记录含对应备注。 |
| 1.3 删除照片 | 编辑时点击某张照片右上角 × | 该张从列表移除，保存后记录中无该图。 |
| 1.4 已达 9 张再添加 | 已有 9 张时点「添加照片」并选图 | 提示「最多 9 张照片」，不新增。 |
| 1.5 咖啡架添加照片 | 咖啡架 → 从相册选多张 | 上传后多张均加入架，每张有唯一 id，可预览、可删除。 |
| 1.6 换设备查看 | 同账号在另一台设备打开 | 新上传的照片（fileID）正常显示；仅存本地路径的旧数据仅本机可见。 |

### 2. 编辑打卡

| 用例 | 操作 | 预期 |
|------|------|------|
| 2.1 进入编辑 | 日历 → 选有记录的日期 → 某条右侧点「编辑」（铅笔） | 进入打卡页，标题「编辑咖啡打卡」，表单为该条内容（来源/店铺/心情/评分/照片）。 |
| 2.2 编辑第一条（index=0）保存 | 对当天第一条点编辑 → 改任意字段 → 今日打卡 | 仅更新该条，不新增一条；提示「已更新打卡」。 |
| 2.3 编辑非第一条保存 | 对当天第二条点编辑 → 改 → 保存 | 仅更新该条，条数不变。 |
| 2.4 编辑中删照片再保存 | 编辑 → 删一张照片 → 保存 | 该条照片数减少，其他条不变。 |
| 2.5 编辑中加照片再保存 | 编辑 → 添加照片 → 填备注 → 保存 | 该条照片增加，新图为 fileID，换设备可见。 |

### 3. 打卡提交与防重复

| 用例 | 操作 | 预期 |
|------|------|------|
| 3.1 消费未填店铺 | 来源选「消费」、店铺为空 → 今日打卡 | 提示「请先填写店铺」，不提交。 |
| 3.2 正常提交 | 填必填项 → 今日打卡 | 按钮出现 loading、不可再次点击；保存成功后清空表单并返回或跳转。 |
| 3.3 云端失败 | 断网或云异常时提交 | 提示「已保存到本地，云端同步失败」，不新增一条；按钮恢复可点。 |

### 4. 日历与删除

| 用例 | 操作 | 预期 |
|------|------|------|
| 4.1 删除一条 | 某日详情 → 某条点删除（垃圾桶） | 该条消失，该日条数减 1；若该日无剩余条则日期不再有标记。 |
| 4.2 编辑入口位置 | 查看某条卡片 | 编辑（铅笔）在删除（垃圾桶）左侧。 |

### 5. 边界与兼容

| 用例 | 操作 | 预期 |
|------|------|------|
| 5.1 无网络/云不可用 | 添加照片或保存 | 照片上传失败或云端同步失败有明确提示，不崩溃。 |
| 5.2 旧数据（本地 path） | 展示含 wxfile:// 等旧 path 的记录 | 本机仍能显示（兼容）；新加的均为 fileID。 |
| 5.3 今日打卡无日期 | 从 tab 直接进打卡页（无 date 参数） | 按今日日期加载，可新增一条。 |

## 三、代码层面已覆盖点

- **editIndex 为 0**：用「number 且 ≥0」判断，避免 `0 || -1` 导致编辑第一条被当成新增。
- **咖啡架 id**：同批多张用 `base + index + random` 保证唯一。
- **防重复提交**：`punching` + 按钮 disabled/loading + 失败时重置。
- **备注**：无弹窗，选图后直接上传，备注在列表内输入框填写并 `onPhotoRemarkInput` 写回。

## 四、建议回归场景（按优先级）

1. 编辑「当天第一条」记录并保存 → 仅更新、不新增。
2. 连续快速点击两次「今日打卡」→ 只写入一条。
3. 选多张照片 → 上传 → 在每张下填备注 → 保存 → 日历/详情中显示正确。
4. 咖啡架一次添加 3 张 → 架上出现 3 张，id 不重复，可单独删除。

完成以上即可认为本次改动通过自测。

## 五、自动跑一遍（逻辑自测，不依赖微信）

在项目根目录执行（需安装 Node）：

```bash
node scripts/run-qa-logic.js
```

脚本用纯逻辑复现：编辑替换/新增、editIndex=0 判断、防重复提交、照片上限、咖啡架 id、删照片、备注更新、消费未填店铺拦截。全部通过会输出「全部通过」，有失败会打印并 exit(1)。
